## grep
Usage: grep [OPTION]... PATTERNS [FILE]...
Search for PATTERNS in each FILE
**When FILE is "-", read from standard input. With no FILE, read "." if recursive, read "-" otherwise. With fewer than two FILEs, assume "-h"**
- some useful options:
    - -e : use PATTERN for matching
    - -f, --file=FILE: take PATTERNs form FILE
    - -E : 拓展正则表达式
    - -P : Perl兼容正则表达式
    - -i : ignore case
    - -s : suppress error messages
    - -v, --invert-match : select non-matching lines
    - -n : with line numbers
    - -c : print only the count of lines selected in each FILE

一、装甲板识别模型应该做到什么程度

不要把目标定义为模糊的“学会深度学习”，而要拆成四个版本。

V0：跑通完整训练流程

目标：

准备几十到一百张图片；
标注装甲板矩形框；
调用预训练模型；
成功训练；
能在一段比赛录像上推理。

这一阶段不追求精度，只验证：

Python 环境是否正常；
CUDA 是否正常；
数据格式是否正确；
训练、验证、推理流程是否闭环。

当前 Ultralytics 工具链支持在自定义数据集上训练目标检测模型，也提供命令行和 Python 接口，非常适合快速建立第一个基线。

V1：装甲板矩形框检测

模型输出：

装甲板位置；
置信度；
敌我颜色，或者装甲板类别。

先证明神经网络确实比你们当前传统视觉方案更稳定。

但矩形框通常只能告诉你装甲板大致在哪。如果后面需要准确 PnP，矩形框的四个边缘不一定对应装甲板真实角点。

V2：装甲板四角点模型

模型输出：

装甲板四个角点；
装甲板类别；
可选的数字、颜色和大小装甲板信息。

四角点可以直接提供给 PnP。已有 RoboMaster 开源项目采用了装甲板四点检测模型，这类项目可以用于理解数据格式、训练流程和后处理，但不建议直接复制后就结束。

对你的项目，比较合理的最终结构是：

相机图像
   ↓
装甲板检测/四角点网络
   ↓
角点排序与筛选
   ↓
PnP
   ↓
Tracker / EKF
   ↓
弹道补偿
   ↓
目标 yaw、pitch
V3：部署和实车测试

最终验收不能只看训练软件里的 mAP，还要看：

远距离装甲板召回率；
快速运动和运动模糊下是否丢失；
灯光、反光、过曝下是否误识别；
每分钟误检次数；
四角点误差；
PnP 解算后的角度误差；
单帧推理延迟；
实际端到端帧率；
和 Tracker 连接后是否稳定。

模型可以导出为 ONNX、TensorRT 等格式。Ultralytics 当前没有专门的官方 C++ 推理 API，因此实际部署通常是将模型导出，再使用 ONNX Runtime、TensorRT 或其他运行时的 C++ API 加载。

你的路线应该是：

    ↓

    ↓

    ↓

    ↓

四、数据集比模型结构更重要

装甲板模型的效果很大程度取决于数据是否贴近真实部署环境。

数据来源优先级
你们自己的相机和镜头；
你们自己的曝光、分辨率和帧率；
训练场和比赛场录像；
队内历史录像；
公开 RoboMaster 数据集。

公开数据可以用于起步，但最终一定要加入自己设备采集的数据。

推荐的数据迭代方式
第一次：50～100 张

目的只是验证标签、训练和推理代码。

第二次：500～1000 张有效图片

覆盖：

不同距离；
不同角度；
不同亮度；
不同背景；
静止和运动；
清晰和模糊；
完整和遮挡装甲板。
第三次：根据错误补数据

不要只是继续随机抽帧，而要专门采集模型容易失败的情况：

很远的装甲板；
装甲板出现在图像边缘；
强反光；
灯条断裂；
车辆高速旋转；
多个装甲板同时出现；
红蓝灯、场地灯等干扰；
没有装甲板的纯负样本。

CVAT 支持 YOLO 的矩形框数据格式，也支持 Ultralytics 的检测、旋转框、分割和姿态格式，可以用来管理后续的矩形框或关键点标注。

数据划分要避免一个常见错误

不要把同一段视频的相邻帧随机分到训练集和验证集。

例如：

第 100 帧：训练集
第 101 帧：验证集

这两张图几乎相同，验证结果会虚高。

应该按照整段视频、采集日期、场景或比赛划分：

录像 A、B、C → 训练集
录像 D       → 验证集
录像 E       → 测试集
